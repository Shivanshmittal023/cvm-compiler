**CVM++** is a lightweight, end-to-end compiler pipeline and stack-based Virtual Machine built entirely from scratch in C++17. It takes a subset of actual C++ source code (Mini-C++), parses it into an Abstract Syntax Tree (AST), compiles it down to a proprietary bytecode, and executes it on a custom Virtual Machine. 

This project demystifies how high-level code is translated into low-level instructions and executed, heavily inspired by modern bytecode VMs like the JVM or Python's CPython.

## ✨ Features

* **Custom Lexer & Parser:** Hand-written recursive descent parser for a subset of C++ grammar.
* **Proprietary Bytecode Compiler:** Traverses the AST and emits a highly optimized, flat array of bytecode instructions (27 custom opcodes).
* **Stack-Based Virtual Machine:** A high-performance execution engine featuring a value stack, call frames, and an instruction dispatch loop.
* **Rich Language Support:**
  * Variables, Scopes, and Types (`int`, `bool`, `double`, `string`)
  * Control Flow (`if/else`, `while`, `for`)
  * Arithmetic and Logical Operators
  * Functions, Arguments, and Recursion
* **Interactive CLI & REPL:** Includes tools to inspect the Token stream, AST, compiled Bytecode, and step-by-step VM Execution trace.

## 🏗️ Architecture Pipeline

```text
Source Code 
   ↓ (Lexer)
 Tokens 
   ↓ (Parser)
 Abstract Syntax Tree (AST) 
   ↓ (Compiler)
 Bytecode & Constant Pool (Chunk) 
   ↓ (Virtual Machine)
 Output / Result
```

## 🚀 Getting Started

### Prerequisites
* A compiler with **C++17** support (e.g., GCC, Clang, or MSVC)
* **CMake** (version 3.10 or higher)

### Build Instructions

To run this project locally, you will need to clone the repository and compile the source code using CMake. CMake is designed to perform an "out-of-source" build, which means you should create a dedicated `build/` directory to hold all of the generated compilation artifacts without cluttering the source tree.

```bash
# 1. Clone the repository
git clone https://github.com/Shivanshmittal023/cvm-compiler.git
cd cvm-compiler

# 2. Create a separate build directory and navigate into it
mkdir build
cd build

# 3. Generate build files and compile the executable
cmake ..
make -j$(nproc)  # Use -j$(sysctl -n hw.logicalcpu) on macOS
```

### Usage

Run a Mini-C++ file:
```bash
./mycc script.cpp
```

Start the interactive REPL (Read-Eval-Print Loop) to code on the fly:
```bash
./mycc --repl
```

**Developer/Debug Flags:**
You can inspect every stage of the pipeline using CLI flags:
* `--tokens`: Print the token stream generated by the Lexer.
* `--ast`: Print the Abstract Syntax Tree generated by the Parser.
* `--bytecode`: Disassemble and print the compiled bytecode.
* `--trace`: Trace VM execution (shows stack state and instruction pointer step-by-step).
* `--all`: Enables tokens, ast, and bytecode flags.

Example:
```bash
./mycc --all my_program.cpp
```

## 💡 Code Example

CVM++ can execute code like this natively:

```cpp
int factorial(int n) {
    if (n <= 1) {
        return 1;
    }
    return n * factorial(n - 1);
}

int main() {
    int sum = 0;
    for (int i = 1; i <= 100; i++) {
        sum = sum + i;
    }
    
    print(factorial(5)); // Output: 120
    print(sum);          // Output: 5050
    return 0;
}
```
